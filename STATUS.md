# Project Status

**Status:** Research artifact. Day-1 demonstration complete.
**Maintenance:** Not actively maintained for outside use.
**Date:** 2026-05-10

## What this means in practice

- Code is reproducible from a clean checkout (numpy + Python 3.10+).
- All claims in `README.md`, `FINDINGS.md`, and `results.json` have been audited and trace to specific test results.
- The architecture is demonstrated only in toy worlds (up to 30 positions × 12 flavors). Scaling is genuinely unknown.

## What we will and won't do

**We won't:**
- Respond to issues or pull requests on a regular basis.
- Provide support or debugging help.
- Take feature requests.
- Engage in extended public discussion.

**We will (occasionally):**
- Fix bugs in the published code if someone points one out clearly.
- Update results if reproducibility breaks.
- Add follow-up work as separate releases if/when the productize path produces a real demo.

This is one author working under autistic + ADHD constraints that make sustained back-and-forth communication infeasible. The code is published as a research artifact and a timestamped statement of the substrate-identity framing — not as the start of a community project.

## If you want to use this

Fork freely. Cite if you publish. The MIT license imposes no obligations on you. We just can't promise to be responsive.

## If you find a clear bug in the experimental claims

Open an issue with the seed, the file, the line number, and the discrepancy between observed output and the documented number. We'll triage but won't promise turnaround.

## What would change this status

A productize follow-up (substrate-as-identity-layer for LLM agents) that warrants its own repo with active development. If that happens, this repo stays as the citable research artifact and the productize work lives elsewhere.
